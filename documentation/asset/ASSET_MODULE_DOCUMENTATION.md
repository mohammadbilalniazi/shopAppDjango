Asset Module Documentation

1. Module Scope

The asset module provides organization-level financial reporting for the supermarket management system. It collects billing activity, loan records, and inventory/cash positions to generate statements used by dashboards and audit views.

This module is designed for thesis documentation because it demonstrates how operational bills are transformed into accounting reports and how denormalized summary tables support performance-sensitive financial analysis.

Primary files:
- asset/models.py
- asset/views.py
- asset/views_financial.py
- asset/utils.py
- asset/urls.py

2. Core Models

AssetBillSummary
- Summarizes bills per year, organization, counterparty, branch, and bill type.
- Used to build inter-organization ledger summaries and quick transaction roll-ups.

AssetWholeBillSummary
- Provides cross-year aggregates by organization and bill type.
- Supports financial dashboards without querying raw bill tables.

OrganizationAsset
- Tracks the current financial position of an organization.
- Includes inventory_value, cash_on_hand, accounts_receivable, accounts_payable, loans_receivable, loans_payable, total_assets, total_liabilities, equity, total_revenue, total_cost_of_goods_sold, total_expenses, total_losses, and net_profit.
- Automatically recalculates totals on save.

Loan
- Records payable and receivable loans separately.
- Supports tracking of amount paid, amount remaining, interest, due dates, and loan status.

ProfitLossStatement
- Stores updated profit and loss values for an organization.
- Used to persist statement data and support historic reporting.

3. Asset Summary and Calculation Flow

The key calculation function is update_organization_assets(organization) in asset/utils.py. It:
- calculates inventory_value from validated bill and stock data
- computes cash_on_hand from bill payment flows
- derives accounts_receivable and accounts_payable from outstanding SELLING and PURCHASE bills
- aggregates loans via payment/receivement bills and the Loan model
- calculates total_revenue, total_cost_of_goods_sold, total_expenses, and total_losses
- saves OrganizationAsset, which auto-updates total_assets, total_liabilities, equity, and net_profit

This flow demonstrates a hybrid architecture:
- raw transactional bills remain normalized in bill tables
- financial metrics are cached in summary models for fast dashboard rendering
- the system keeps accounting data consistent by recomputing assets after bill changes

4. Financial Statement Generation

Balance Sheet
- Generated by get_balance_sheet(organization).
- Returns current assets, current liabilities, and equity.
- Current assets include cash, accounts receivable, inventory, and loans receivable.
- Total liabilities include accounts payable and loans payable.
- Equity is retained earnings plus net profit.

Profit and Loss Statement
- Generated by get_profit_loss_statement(organization).
- Returns revenue, cost of goods sold, gross profit, operating expenses, losses, and net profit.
- Uses values cached in OrganizationAsset, making statement generation fast and consistent.

Cash Flow Summary
- Generated by get_cash_flow_summary(organization).
- Breaks cash activity into operating activities and financing activities.
- Uses SELLING, PURCHASE, EXPENSE, LOSSDEGRADE, RECEIVEMENT, and PAYMENT bills.
- Computes net cash flow from operating cash and financing cash.

5. Views and Routes

Primary views:
- asset_dashboard: shows asset overview and financial statements
- balance_sheet_view: renders the detailed balance sheet
- profit_loss_view: renders the profit and loss statement
- cash_flow_view: renders cash flow details
- loans_view: shows loan payables and receivables

Financial reporting and ledger views:
- organization_ledger_summary: summary or detail ledger between two organizations
- ledger_adjustment_save: saves manual ledger adjustments
- financial_summary_dashboard: dashboard using AssetWholeBillSummary for performance
- admin_adjust_summary: admin interface for correcting cached summary values

API endpoints:
- api/refresh/ : refreshes asset calculations on demand
- api/summary/<org_id>/ : returns JSON asset summary and statements

6. Thesis Discussion Points

- Denormalized summaries vs normalized bill storage: the asset module uses cached summary tables to balance performance and accuracy.
- Financial statement mapping: the module shows how retail bill types map to balance sheet, profit-loss, and cash flow categories.
- Ledger reconciliation: OrganizationAsset and AssetBillSummary support both summary and detailed ledger views for inter-organization accounting.
- Manual adjustment controls: the ledger adjustment process highlights how real-world systems handle reconciliation differences and audit corrections.
- Multi-organization, branch-aware reporting: the module is designed to support organization-specific and branch-granular financial reporting, which is critical in supermarket chains.
